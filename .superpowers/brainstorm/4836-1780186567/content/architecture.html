Arquitectura de Centralización

¿Dónde debe residir la lógica y los datos de las solicitudes?

A

Centralizado en Nexus (Recomendado)

SyncXML envía la solicitud a Nexus via API. Nexus gestiona validación, persistencia y envío de correos. SyncXML queda como cliente ligero.

SyncXML UI -> Nexus API -> DB (Nexus) -> Resend/Email
B

Local en SyncXML

SyncXML gestiona su propia tabla de solicitudes. Mayor independencia pero duplica lógica de "leads" que ya existe en Nexus.

SyncXML UI -> SyncXML API -> DB (Local) -> Resend/Email

Ventajas Nexus

  • Visión 360º del usuario en el ecosistema Anclora.
  • Reutilización de validaciones IA y SLAs ya existentes.
  • Panel de administración único para todos los productos.

Desafíos

  • Requiere comunicación segura entre servicios.
  • Dependencia de Nexus para el flujo de alta.
