# Security Policy

## Supported status

Anclora SyncXML is in pre-MVP / controlled validation.

## Reporting a vulnerability

Please do not open public issues for vulnerabilities involving:

- Authentication bypass.
- Secret leakage.
- PII exposure.
- XML injection.
- Unsafe file upload handling.
- Storage or encryption issues.

Report privately to:

`pmi140979@gmail.com`

## Data handling rules

- Do not submit real guest data in issues, discussions or pull requests.
- Use synthetic or anonymized files only.
- Remove PII from logs and screenshots.

## Security scope

Areas of particular interest:

- Authentication and session handling.
- File upload validation.
- XLSX parsing.
- XML generation and validation.
- Blocking DOCTYPE / ENTITY.
- Encryption and retention.
- Audit logs without PII.
- Rate limiting.

## Responsible disclosure

Acknowledge receipt within a reasonable timeframe and avoid public disclosure
until a fix or mitigation exists.
