# Knowledge Packs — Anclora SyncXML

Esta carpeta almacena manifiestos y paquetes documentales preparados para actualizar una futura Gema/GPT de Anclora SyncXML.

La Gema/GPT no debe contener todas las fuentes brutas. Debe alimentarse con documentos consolidados, auditados y accionables.

## Objetivo

Organizar qué documentos deben usarse para crear o actualizar una Gema/GPT de apoyo estratégico, técnico y de producto para Anclora SyncXML.

## Incluir preferentemente

- diagnóstico refinado;
- matriz de prioridades;
- plan de acción;
- registro de riesgos;
- índice maestro;
- changelog ejecutivo;
- resumen estratégico del mercado;
- auditoría de competidores;
- auditoría de posicionamiento;
- posicionamiento validado;
- documentos consolidados de privacidad, responsabilidad y roadmap.

## Evitar

- snapshots completos antiguos;
- repo completo en formato .txt, salvo que la Gema/GPT vaya a revisar código;
- informes duplicados;
- presentaciones;
- infografías;
- versiones iniciales sustituidas por versiones refinadas;
- fuentes comerciales no auditadas;
- documentos con afirmaciones no verificadas.

## Regla principal

La Gema/GPT debe trabajar con conocimiento consolidado, no con todas las fuentes brutas.

NotebookLM conserva las fuentes profundas.  
La Gema/GPT debe usar documentos refinados para ayudar a decidir, priorizar, redactar y ejecutar.

## Convención de versiones

Usar este formato para manifiestos:

KNOWLEDGE_PACK_vX.Y-descripcion_MANIFEST.md

Ejemplos:

- KNOWLEDGE_PACK_v0.1-baseline_MANIFEST.md
- KNOWLEDGE_PACK_v0.2-mvp_MANIFEST.md
- KNOWLEDGE_PACK_v0.3-precheckin_MANIFEST.md
