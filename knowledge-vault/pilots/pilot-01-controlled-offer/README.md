# Pilot 01 — Controlled Offer

Este directorio contiene los documentos operativos del primer piloto controlado de Anclora SyncXML.

El objetivo del piloto no es lanzar el producto ni trabajar con datos reales, sino validar de forma controlada el flujo:

Excel/XLSX → revisión → detección de errores → XML revisable

## Reglas del piloto

- Usar datos sintéticos, anonimizados o muestras controladas.
- No usar datos reales de huéspedes salvo cierre previo de RGPD, DPA, retención, seguridad, validación técnica y responsabilidades.
- No prometer cumplimiento legal.
- No prometer evitar sanciones.
- No afirmar integración oficial con SES.HOSPEDAJES.
- No afirmar envío automático al Ministerio del Interior.
- No sustituir PMS, gestoría ni asesoramiento legal.
- Documentar riesgos y decisión go/no-go antes de cualquier contacto externo.
