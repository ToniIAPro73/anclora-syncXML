# CLAIMS_AND_LIMITS_PILOT_v0.1 — Claims y límites comerciales del piloto

## Estado
Borrador inicial / v0.1

## Propósito
Establecer un marco de comunicación seguro para el equipo comercial y de producto. El objetivo es evitar generar expectativas falsas o asumir responsabilidades legales que el software aún no debe cubrir.

## Principio general
Anclora SyncXML debe comunicarse como un **asistente técnico de preparación y revisión**, nunca como una "solución de cumplimiento legal" o una "integración oficial".

## Claims seguros (Recomendados)

| Claim | Uso recomendado | Motivo |
| :--- | :--- | :--- |
| "Revisa tus datos de huéspedes desde Excel/XLSX." | Principal | Es la funcionalidad core verificada. |
| "Detecta errores operativos antes de generar tu XML." | Enfoque en ahorro de tiempo | Pone el valor en la prevención de errores humanos. |
| "Genera un XML revisable de prueba para tu flujo SES." | Descripción técnica | Usa "revisable" y "de prueba" para evitar compromiso de aceptación. |
| "Herramienta ligera para alojamientos que usan Excel." | Posicionamiento | Define el nicho de mercado sin prometer complejidad de PMS. |
| "Validación controlada con datos sintéticos." | Transparencia técnica | Refuerza la seguridad y la fase actual del proyecto. |

## Claims prudentes con matices (Usar con precaución)

| Claim | Riesgo | Matiz obligatorio | Versión recomendada |
| :--- | :--- | :--- | :--- |
| "Orientado al flujo SES.HOSPEDAJES." | Sugerir compatibilidad total. | "Basado en la estructura técnica conocida." | "Prepara tus datos para el flujo de SES." |
| "Validación de datos de huéspedes." | Sugerir validación legal/oficial. | "Validación según reglas lógicas internas." | "Asistente de validación operativa." |
| "XML para SES." | Sugerir que está validado por el Ministerio. | "XML revisable de prueba." | "Genera el XML orientado a SES." |
| "Privacidad por defecto." | Sugerir auditoría RGPD completa. | "En el marco del piloto controlado." | "Diseñado bajo principios de minimización." |

## Claims arriesgados (Evitar)

| Claim | Riesgo | Alternativa segura |
| :--- | :--- | :--- |
| "Compatible con SES.HOSPEDAJES." | Implica certificación que no existe. | "Genera archivos siguiendo la estructura SES." |
| "Cumplimiento normativo SES." | Asume responsabilidad legal. | "Ayuda a preparar los datos para la norma." |
| "Reduce el riesgo de sanciones." | Promesa de resultado legal. | "Ayuda a detectar errores antes del envío." |
| "Automatiza tu gestión de viajeros." | Sugerir que no requiere intervención. | "Agiliza la revisión de tus viajeros." |

## Claims prohibidos (Terminantemente prohibidos)

| Claim prohibido | Por qué está prohibido | Alternativa |
| :--- | :--- | :--- |
| "Cumplimiento legal garantizado." | El software no da asesoría legal. | "Asistente técnico de revisión." |
| "Evita multas y sanciones." | Riesgo jurídico extremo. | "Detecta errores comunes en el Excel." |
| "Integración oficial con el Ministerio." | No existe tal convenio técnico. | "Orientado al flujo técnico oficial." |
| "Envío automático al Ministerio/SES." | La funcionalidad no está activa/probada. | "Preparación de archivo para subida manual." |
| "XML aceptado por SES." | La aceptación depende de SES, no de nosotros. | "XML revisable según esquema." |
| "Sustituye a tu gestoría/asesoría." | Intrusismo y riesgo de mala praxis. | "Complemento para tu gestión interna." |

## Mensaje recomendado largo (Propuesta/Web)
"Anclora SyncXML es un asistente técnico en fase de validación controlada diseñado para simplificar la revisión de datos de huéspedes. Nuestra herramienta permite a pequeños propietarios importar sus hojas de Excel, detectar errores operativos de forma visual y generar un XML revisable orientado a la estructura de SES.HOSPEDAJES. Durante este piloto, trabajamos exclusivamente con datos de prueba para validar el ahorro de tiempo y la precisión técnica, sin sustituir en ningún caso las obligaciones legales del titular."

## Mensaje recomendado corto (Elevator pitch)
"Ayudamos a pequeños alojamientos a limpiar sus Excel de huéspedes y preparar el XML de viajeros de forma rápida y sin errores técnicos, antes de su comunicación oficial."

## Disclaimer obligatorio
*"Anclora SyncXML es una herramienta de soporte técnico pre-MVP. No constituye asesoramiento legal ni garantiza la aceptación de los archivos por parte de las autoridades. El usuario es el único responsable de la veracidad de los datos y del cumplimiento de sus obligaciones legales."*

## Checklist antes de enviar material comercial
- [ ] ¿He evitado la palabra "Garantizado"?
- [ ] ¿He evitado la palabra "Oficial"?
- [ ] ¿He dejado claro que el piloto usa datos sintéticos?
- [ ] ¿He definido el producto como "Asistente" o "Herramienta de revisión"?
- [ ] ¿He incluido el disclaimer obligatorio al final?
