# Support

## Project status

Anclora SyncXML is in pre-MVP / controlled validation.

Support should reflect that reality: maintainers may prioritize correctness,
privacy and security over speed or feature breadth.

## Where to ask for help

- Use GitHub Issues for reproducible bugs and well-scoped feature requests.
- Use documentation issues or pull requests for onboarding and docs fixes.
- Use the security contact in [SECURITY.md](SECURITY.md) for sensitive reports.

## Before opening an issue

- Confirm the problem still exists on the latest `main`.
- Include reproduction steps, expected behavior and actual behavior.
- Share relevant environment details.
- Remove secrets and all guest PII from logs, screenshots and attachments.

## What maintainers can realistically help with

- Setup and contributor onboarding problems.
- Bugs that can be reproduced from the repository state.
- Documentation gaps.
- Privacy-first and XML-validation design questions.

## What support does not cover

- Legal advice.
- Compliance certification.
- Review of real guest datasets.
- Emergency production support or SLA-backed response times.

## Security and privacy

If your report involves authentication bypass, secret leakage, PII exposure,
unsafe uploads, XML injection or storage/encryption weaknesses, do not post it
publicly. Follow [SECURITY.md](SECURITY.md).
