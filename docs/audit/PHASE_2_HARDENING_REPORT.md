## Implemented Changes

### Villa Kentia Excel Parser

The existing parser already segments the Villa Kentia workbook into guest rows and lower metadata. The regression suite verifies that `test-data/fixtures/registro_huespedes_synthetic.xlsx` produces:

- 7 guests.
- Establishment code `0000044116`.
- Reference `5992657522`.
- Guest count `7`.
- Payment type `PLATF`.
- Check-in `2026-04-30`.
- Check-out `2026-05-03`.

No additional parser rewrite was required for this phase.

### Authentication and Sensitive Route Hardening

Added in-memory rate limiting for:

- `POST /api/auth/login`
- `POST /api/upload/excel`
- `POST /api/generate/xml`
- `POST /api/reservations`

The production authentication gate remains deny-by-default:

- Missing `SYNCXML_ADMIN_PASSWORD` blocks password auth.
- Missing `SESSION_SECRET` or `AUTH_SECRET` blocks password auth.
- Local demo mode is only allowed outside production with explicit `SYNCXML_LOCAL_DEMO=true`.

### Backend Validation Before Consolidation

`POST /api/reservations` no longer trusts a client-provided `generated` payload. It now:

1. Authenticates the request.
2. Applies rate limiting.
3. Revalidates parsed data server-side with `smartValidateParsedExcel`.
4. Regenerates XML server-side from the validated payload.
5. Blocks critical validation errors.
6. Blocks unresolved duplicates.
7. Persists the server-generated XML only after those checks.

`POST /api/generate/xml` also runs server-side smart validation before XML generation.

### XML Persistence and Download

Persistent reservations now include a `generatedXml` column:

- Prisma field: `Reservation.generatedXml`.
- Migration: `prisma/migrations/20260523170000_add_generated_xml_to_reservation/migration.sql`.

The XML download route now returns the stored XML generated by the application. If no XML exists for a reservation, the route returns a clear `404` JSON error instead of fabricating an XML error document.

### AES-256-GCM Encryption Roundtrip

The encryption helper now exposes `decryptString` for fields encrypted with `encryptString`. The regression test verifies:

- Encrypted values use the `enc:v1:` envelope.
- Decryption returns the original value.
- Encryption remains non-deterministic for buffers.

### Timezone Correctness

XML date generation no longer hardcodes `+02:00`. It now calculates the `Europe/Madrid` offset for the relevant date:

- Summer dates use `+02:00`.
- Winter dates use `+01:00`.

This is appropriate for Mallorca/Villa Kentia and avoids generating winter XML with the wrong offset.

### Placeholder Detection

The placeholder detector no longer blocks any legitimate reservation containing the date `2026-04-09`. It now detects the specific example timestamp from the template context instead of using a broad date substring match.
